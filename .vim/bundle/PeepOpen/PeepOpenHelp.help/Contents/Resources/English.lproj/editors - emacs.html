Emacs Plugin and Tips


PeepOpen works with several flavors of Emacs. 


• Carbon Emacs http://homepage.mac.com/zenitani/emacs-e.html

• Aquamacs Emacs http://aquamacs.org/


We use Emacs to write nearly all our software, from web apps to Cocoa desktop and mobile apps.


Plugin


Go to the PeepOpen Preferences menu and choose Emacs as your editor. Click the Install Plugin button.



Now, go to ~/.emacs.d/vendor/peepopen.el and read the comments. You’ll need to install Chris Wanstrath’s textmate.el plugin and edit your Emacs configuration file to require both files.


We now include configuration files for Aquamacs. Run Install Plugin and look at Preferences.sample.el that we copied to your ~/Library/Application Support/Aquamacs Emacs directory.


Usage


  • Navigate to a directory or a file in Emacs. 
  • Hit ⌘-T to activate PeepOpen. 
  • Search for a file and hit Enter to open it in Emacs.


The project-defining functionality is handled by textmate.el. It looks up the directory structure for a .git or .hg directory, or a Rakefile and considers that to be the root of the current project.


Tips


You can switch between several projects in the current frame and textmate.el will recognize them as individual projects. The PeepOpen window will include only the files in the current project being edited.


Aquamacs and other flavors of Emacs may need a special directive to open files in the existing frame (instead of spawning a new one every time). Try this snippet:

Uninstall


Remove ~/.emacs.d/vendor/peepopen.el and remove the corresponding lines in your Emacs configuration file.
